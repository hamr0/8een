8een — replay defense, live

The same proof is accepted once, then refused on replay. That refusal is the single-use nonce firing — real longfellow verification underneath.

1

Get a proof from the wallet

The site asks 8een for a one-time nonce; the wallet builds a proof stamped with it. (Here the “wallet” is a pre-minted pool — the real one is a phone.)

2

Verify it

POST the proof to the real gate at /8een/verify. First use of the nonce → accepted, nonce burned.

3

Replay the exact same bytes

Re-POST the identical proof — what an attacker who recorded it would do. The nonce is already spent → refused.

ok:false is never “underage.” A replay comes back ok:false, over_threshold:null, reason:replay_detected — “I can’t confirm this is fresh, ask again,” not a verdict about the person. Deny entry, but never tell them they failed the age check.
